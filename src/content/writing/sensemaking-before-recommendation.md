I am more interested in sensemaking tools than recommendation engines.

Recommendation is seductive because it gives the user an answer. Sensemaking is slower. It gives the user a surface where the evidence, categories, uncertainty, and tradeoffs can be inspected.

Most of the tools I keep building sit in that second space.

CoinGossip tried to read attention around cryptocurrencies, not just price. Now In Singapore gathers scattered event listings and turns them into a browsable pulse-check. Skills Framework Explorer makes a dense skills taxonomy easier to navigate. AI-assisted research prototypes sit in the same family: they are only useful if the person can see what the system read, what it ignored, and how the output was shaped.

## The shape of the tool matters

If the tool hides the signal, the user has to trust the conclusion.

That is sometimes fine for low-stakes convenience. It is not enough when the output informs policy, career, research, investment, or organisational decisions. In those cases, the interface has to preserve a trail back to source material, categories, filters, and assumptions.

The point is not to make every user read everything. The point is to let judgement stay in the loop.

This is why I tend to build tools that collect, summarise, categorise, and expose. The recommendation can come later. First, show me the terrain.
