# Brand Toolkit

## Premise

Jin's site is a personal systems map with evidence trails. It should feel like a well-kept workshop near water: calm, structured, useful, and personal.

## Reference Sheet

The generated visual toolkit is stored at `docs/design-references/brand-toolkit-reference-sheet.png`.

Use it as a directional reference, not a literal implementation spec.

## Palette Direction

Use a committed green palette carried by muted, natural greens and river tones:

- Moss: primary identity color for section anchors, active states, and structural lines.
- River Teal: secondary signal for links, route highlights, and connected trails.
- Sage: quiet supporting surfaces and hover states.
- Warm Paper: primary page background, never pure white.
- Ink Green-Gray: main text color, never pure black.
- Soft Signal: restrained emphasis for tags, status, and annotations.
- Clay Note: occasional warmth for human notes, shelf items, and reflective details.

Use OKLCH values in implementation. Keep chroma controlled at very high and very low lightness.

## Typography Direction

Choose type that feels functional, humane, and precise. Avoid default portfolio typography and avoid over-editorial display-serifs.

Recommended system:

- Headings: refined sans-serif with strong weight contrast and calm proportions.
- Body: readable humanist sans, optimized for long-form writing.
- Annotation style: optional compact sans for labels, tags, dates, and side notes.

Do not use monospace as shorthand for technical credibility. Use it only if a specific code or terminal affordance requires it.

## Layout Grammar

- Build around a systems map, not a hero plus card grid.
- Use evidence trails: threads connect to work, writing, projects, and influences.
- Let section rhythm vary: map-like homepage, index-like Workbench, calm long-form Writing.
- Keep body text to 65-75ch.
- Use precise spacing and visible structure without making everything a card.

## Interaction Grammar

- Hover or tap reveals connected projects, posts, jobs, or influences.
- Mobile collapses the systems map into a clear stacked index.
- Reduced motion preserves all information without animated dependency.
- Motion should feel like line drawing, revealing, or quiet routing, not spectacle.

## Voice

Lead with plain credibility:

- Say what Jin is working on.
- Show the terrain he has worked in.
- Let evidence trails carry details, metrics, and receipts.
- Preserve project archaeology where it reveals recurring patterns of noticing friction and building systems around it.

Avoid inflated AI language, generic personal branding, and resume-like achievement blocks.

## Do / Don't

Do:

- Calm map.
- Specific receipts.
- Readable writing.
- Project archaeology.
- Maker signal.
- Humane systems framing.

Don't:

- LinkedIn cards.
- Neon AI motifs.
- Consultant gloss.
- Chaotic maximalism.
- Generic portfolio grids.
- Decorative hacker styling.
